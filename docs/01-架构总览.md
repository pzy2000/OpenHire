# OpenHire 架构总览

## 概述

OpenHire 是数字员工编排平台本体（Python 包名 `openhire`）。通过 IM 群聊（飞书、Telegram 等）一键拉入数字员工，自动为团队成员分配 AI 分身，每个分身拥有独立的角色、技能、工具和记忆。

## 三层架构

```
+-------------------------------------------------+
|  IM 层 (飞书/Telegram/Discord 群聊)              |
|  统一 bot 入口，群成员看到的是一个"团队"           |
+-------------------------------------------------+
|  OpenHire 编排层                                 |
|  - 数字员工注册/分配/销毁/合并                     |
|  - 角色 → 技能/工具 映射                          |
|  - 消息智能路由                                   |
|  - 跨项目记忆迁移                                 |
+-------------------------------------------------+
|  执行层 (Docker Adapter 框架)                     |
|  openclaw(ACP) / nanobot / hermes                |
|  每个 subagent 独立容器，独立上下文                |
+-------------------------------------------------+
```

## 核心模块

| 模块 | 路径 | 职责 |
|------|------|------|
| Docker Adapter | `openhire/adapters/` | 容器化 agent 执行框架 |
| Agent 注册表 | `openhire/workforce/registry.py` | 数字员工身份和成员管理 |
| 消息路由 | `openhire/workforce/router.py` | 群消息智能分发 |
| 生命周期 | `openhire/workforce/lifecycle.py` | 创建/销毁/挂起/合并 |
| 持久化存储 | `openhire/workforce/store.py` | JSON 文件存储 |
| OpenHire Tool | `openhire/workforce/tool.py` | LLM 可调用的管理工具 |

## 消息流转

1. 飞书群消息进入 OpenHire 主 agent。
2. `MessageRouter.route()`：`@` 某人、关键词、LLM 分类或回退主 agent。
3. `docker_agent` 工具在对应容器内执行。
4. 结果回到群聊。

## 配置

```json
{
  "openhire": {
    "enabled": true,
    "autoRoute": true,
    "llmRouteThreshold": 0.7
  },
  "tools": {
    "dockerAgents": {
      "enabled": true,
      "agents": {
        "openclaw": {
          "persistent": true,
          "acp": {
            "defaultAgent": "claude",
            "allowedAgents": ["claude", "codex", "opencode", "gemini"]
          }
        }
      }
    }
  }
}
```

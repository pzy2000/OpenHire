# 数字员工管理

## 概述

OpenHire 编排层管理数字员工的完整生命周期：注册、分配、路由、挂起、恢复、合并、销毁。每个数字员工绑定一个真人，拥有独立的角色、技能、工具和 Docker 容器。

## 数字员工数据结构 (AgentEntry)

| 字段 | 说明 | 示例 |
|------|------|------|
| agent_id | 唯一 ID | "e15e430d" |
| name | 显示名 | "前端小王的数字员工" |
| owner_id | 绑定的真人 ID | "user_wang" |
| role | 角色描述 | "资深前端工程师" |
| agent_type | adapter 类型 | "openclaw" |
| skills | 技能标签 | ["react", "typescript", "css"] |
| tools | 工具列表 | ["github", "figma"] |
| acp_agent | ACP 底层 agent | "claude" |
| group_ids | 所属群列表 | ["group_123", "group_456"] |
| container_name | Docker 容器名 | "openhire-e15e430d" |
| memory_namespace | 记忆命名空间 | "e15e430d" |
| status | 状态 | "active" / "suspended" / "archived" |

## Agent 注册表 (AgentRegistry)

存储路径：`workspace/openhire/agents.json`

### CRUD 操作

```python
registry.register(AgentEntry(...))     # 注册
registry.get("agent_id")              # 查询
registry.update("agent_id", role=...) # 更新
registry.remove("agent_id")           # 删除
```

### 查询

```python
registry.by_owner("user_wang")        # 按人查
registry.by_group("group_123")        # 按群查
registry.by_skill("react")            # 按技能查
registry.get_group_roster("group_123") # 群花名册
```

### 群管理

```python
registry.join_group("agent_id", "group_123")   # 加入群
registry.leave_group("agent_id", "group_123")  # 离开群
```

## 生命周期管理 (AgentLifecycle)

### 创建数字员工

```python
await lifecycle.create_agent(
    name="前端小王",
    owner_id="user_wang",
    role="资深前端工程师",
    agent_type="openclaw",
    skills=["react", "typescript"],
    tools=["github"],
    acp_agent="claude",
    group_id="group_123",
)
```

### 群项目管理

```python
# 新群/新项目：批量分配
await lifecycle.setup_group("group_123", members=[
    {"owner_id": "user_wang", "name": "前端小王", "role": "前端", "skills": ["react"]},
    {"owner_id": "user_li", "name": "后端小李", "role": "后端", "skills": ["python", "api"]},
])

# 项目结束：批量清理
await lifecycle.close_group("group_123", archive=True)
```

### 合并

```python
# 将 source 的技能、工具、群成员合并到 target
await lifecycle.merge_agents(source_id="abc", target_id="def")
```

### 状态流转

```
创建 → active
         ├── suspend → suspended → resume → active
         ├── destroy → archived (保留记忆) 或 彻底删除
         └── merge → source archived, target 获得合并技能
```

## 消息路由 (MessageRouter)

### 路由优先级

1. **@ 路由** — 消息中 @ 了某人，路由到该人的数字员工
2. **技能匹配** — 关键词匹配 agent 的 skills 标签
3. **LLM 分类** — 用 LLM 做意图分类（需配置 provider）
4. **无匹配** — 交给主 agent 自行处理

### 技能匹配规则

- 直接匹配：消息中包含 agent 的 skill 标签 → +2 分
- 工具匹配：消息中包含 agent 的 tool 名称 → +2 分
- 关键词扩展：如"前端"匹配 ["react", "vue", "css", "typescript"] → +1 分
- 取最高分的 agent（可能多个并列）

### 路由示例

```
消息: "帮我改一下 React 组件的样式"
  → 技能匹配: "react" 命中前端 agent, "css" 也命中
  → 路由到: 前端小王

消息: "数据库查询太慢了，帮我优化一下 API"
  → 技能匹配: "database" + "api" 命中后端 agent
  → 路由到: 后端小李
```

## OpenHire Tool

LLM 可通过 `openhire` tool 管理数字员工：

| Action | 说明 | 必需参数 |
|--------|------|---------|
| register | 注册新数字员工 | name, owner_id |
| list | 列出所有数字员工 | - |
| get | 查询详情 | agent_id |
| update | 更新信息 | agent_id + 字段 |
| destroy | 销毁 | agent_id |
| suspend | 挂起 | agent_id |
| resume | 恢复 | agent_id |
| merge | 合并 | agent_id, target_id |
| setup_group | 初始化群 | group_id, members |
| close_group | 关闭群 | group_id |
| group_roster | 群花名册 | group_id |
| route | 消息路由 | group_id, message |

openhire/workforce/skill_selection.py

3. 优秀员工协议 · 默认技能

所有数字员工自带、禁止删除；7 条协议点击即可展开完整正文

日报协议
每天固定格式总结

每天固定总结：

  • 完成：已经交付的事项和可验证结果
  • 进行中：正在推进的事项、当前阶段和预计下一步
  • 阻塞：等待谁、缺什么、卡在哪里
  • 明日计划：下一天最重要的 1 ~ 3 件事
周报协议
每周汇总 + 决策请求

每周汇总：

  • 成果：本周完成的关键交付
  • 风险：可能影响目标的风险、异常和趋势
  • 决策请求：需要真人判断、授权或拍板的事项
  • 下周重点：下周优先级、预期产出和需要协作的人
巡检协议
主动发现异常
定时检查自己负责的系统、文档、群聊、任务池和外部反馈。发现长期未闭环、状态异常、信息过期或风险升高时，主动提醒并给出下一步建议。
交接协议
任何任务随时可交接
任何长期任务都维护可交接状态，至少包含背景、目标、当前进度、已尝试路径、关键链接、阻塞点和下一步。别人接手时不需要从零翻聊天记录。
证据协议
结论必须可验证
重要结论必须附来源或验证方式。涉及代码、配置、数据、客户反馈、招聘流程或运营判断时，优先给出链接、文件、日志、截图、测试结果或可复现步骤。
升级协议
及时升级给真人
遇到权限、成本、安全、生产风险、需求冲突、信息不足或无法验证的关键判断时，及时升级给真人。升级时说明原因、影响、候选方案和推荐选择。
协作协议
多员工参与时的噪音控制
需要多个数字员工参与时，由主员工聚合结论，避免在群聊里制造噪音。协作输出必须包含责任人、任务边界、交付物、截止时间和最终汇总。
HOW IT IS WIRED

所有员工的 system prompt 会在头部注入一个包裹块，确保 7 条协议在每次对话前被 LLM 读到：

[OpenHire Required Skill:
 excellent-employee]
 ... 7 条协议正文 ...
[/OpenHire Required Skill:
 excellent-employee]

openhire/workforce/required_skill.py

GUARANTEES
  • 禁止删除：调用 delete API 会抛 RequiredSkillDeleteError
  • 创建员工时自动补齐，即使用户未勾选
  • 内容编辑后可一键同步已在编员工
  • admin 列表有 必装 视觉标记
Mentor 视角：协议 ≠ 岗位技能。把这层「工作方式」协议做成默认能力，等价于给所有数字员工发了一本 员工手册，让他们天然具备稳定汇报、主动巡检、可交接、带证据、会升级、会协作的底层习惯。

测

测试总监 · 测试与验收预研

需求核心不是「做一个看板页面」，而是「稳定、准确、可追溯地完成采集 / 清洗 / 展示 / 告警 / 导出」。测试重点从前端展示前移到数据链路正确性、监控时效性、告警可信度、结果可审计性。

查看完整会话 (openhire-delegate-openhire-a60e0f05.jsonl)

质量目标：采集成功率 ≥ 98% · 价格匹配准确率 ≥ 99% · 告警触达延迟 ≤ 5 分钟。

风险场景：结构漂移 / 促销标签混淆 / 库存状态口径不一致 / 告警误报。
